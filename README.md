# 불법광고물 정비 촬영 앱

## 작동 방식
- 휴대폰에서 `전 사진 촬영`
- 촬영 즉시 Google Drive의 `불법광고물 정비사진` 폴더에 업로드
- 화면이 자동으로 `후 사진 촬영`으로 전환
- 후 사진 업로드 후 다음 건의 전 사진 촬영 상태로 복귀
- 장소·내용·저장 버튼 없음

## 1. Google Apps Script 서버 만들기
1. https://script.google.com 에서 **새 프로젝트**를 만듭니다.
2. 기본 `Code.gs` 내용을 모두 지우고, 이 묶음의 `Code.gs` 내용을 붙여넣습니다.
3. 오른쪽 위 **배포 → 새 배포 → 유형 선택 → 웹 앱**을 선택합니다.
4. 실행 사용자: **나**
5. 액세스 사용자: 촬영자가 접속할 수 있는 범위로 설정합니다.
   - 개인 Google 계정은 보통 `모든 사용자`
   - 기관 계정은 관리자 정책 때문에 외부 공개가 막힐 수 있습니다.
6. 배포하고 권한을 승인한 뒤 `/exec`로 끝나는 웹 앱 주소를 복사합니다.

## 2. 앱에 서버 주소 넣기
`index.html`을 메모장으로 열고 아래 문구를 찾습니다.

PASTE_YOUR_APPS_SCRIPT_WEB_APP_URL_HERE

그 자리에 복사한 Apps Script `/exec` 주소를 넣고 저장합니다.

## 3. 무료로 설치 가능한 주소 만들기
이 폴더의 파일 전체를 GitHub 저장소에 올린 뒤 GitHub Pages를 켭니다.

- 저장소 Settings → Pages
- Deploy from a branch
- `main` / `(root)` 선택
- 생성된 `https://아이디.github.io/저장소명/` 주소를 휴대폰 Chrome에서 엽니다.

## 4. 안드로이드 휴대폰에 설치
Chrome 메뉴(⋮) → **앱 설치** 또는 **홈 화면에 추가**

## 주의
- 인터넷이 끊긴 상태에서는 사진 업로드가 되지 않습니다.
- 앱은 전·후 사진을 촬영 순서로 묶습니다. 중간에 촬영을 취소해도 상태는 바뀌지 않습니다.
- `fetch(..., mode:"no-cors")` 방식이라 화면에서는 서버의 최종 저장 응답을 읽지 못합니다. 실제 시험 때 Google Drive 폴더에 사진이 생겼는지 확인하십시오.
- 기관 Google Workspace 정책에 따라 Apps Script 웹앱 외부 접근이나 Drive 저장 권한이 제한될 수 있습니다.
